# 《星辰：物质的熔炉》玩法映射

- 原典来源：Evolution-《第1章 第5篇 星辰：物质的熔炉》
- 规则包：`content/stellar/presets/first-light.json`
- 状态 schema：StellarSpec / StellarState v1
- 当前目标：用一个短时、可保存、可重放的纵切验证完整恒星生命周期

## 1. 改编边界

本纵切保留原典中的五个核心事实：

1. 聚集推动新的聚集；
2. 星辰依靠向内聚集与向外释放的动态平衡延续；
3. 星光把内部变化带向远方；
4. 星辰内部产生更复杂的物质结构；
5. 星辰死亡不是清零，而是向下一代星体系统播种。

原典没有给出物理单位、固定持续时间和唯一爆发结果。因此规则使用抽象安全整数表达相对关系，不把现实天体物理常数或“32维”硬编码成线性关卡，也不把文明实验倒写为正典。

## 2. 生命周期

```text
nebula 物质云
  → protostar 原恒星
  → main_sequence 稳定燃烧
  → red_giant 衰老与外层扩张
  → supernova 核心坍缩和爆发
  → planetary_disk 初生行星盘
```

每个阶段只由前一 tick 的权威状态、StellarSpec 和保存于状态中的 PRNG 推进。表现客户端只能提交时间控制，不能直接设置质量、温度、光度或行星盘稳定度。

## 3. 权威指标

| 指标 | 含义 | 主要表现用途 |
| --- | --- | --- |
| `nebulaMass` | 尚未进入星体的物质 | 星云粒子数量与吸积范围 |
| `stellarMass` | 聚合核心及恒星质量 | 星体尺度和坍缩强度 |
| `corePressure` | 向内聚集形成的核心压力 | 核心脉动与点燃前反馈 |
| `temperature` | 内部转化强度 | 星体颜色与扰动速度 |
| `luminosity` | 向外释放强度 | 发光、辉光和场景曝光 |
| `stability` | 聚集与释放的动态平衡 | 表面震荡幅度 |
| `fuel` | 稳定燃烧阶段的可用燃料 | 主序阶段进度 |
| `elementDiversity` | 已形成的复杂物质层级 | 色彩、物质云和结局摘要 |
| `expelledMatter` | 已离开星体的物质 | 外层抛射与爆发粒子 |
| `diskMass` / `diskStability` | 被下一代系统保留的物质和结构度 | 行星盘密度与最终里程碑 |

`diskMass` 是 `expelledMatter` 中被恒星系统保留的子集，不能与总质量守恒式重复相加。

## 4. 时间控制与重放

`StellarController` 初始暂停，提供 `pause`、`resume`、`step`、`set_speed`、`advance` 和 `pulse`。每次修改都使用 revision 乐观并发。记录恢复时从 spec 与 seed 重新运行到保存 tick，并逐字段比较状态；无法重放的记录失败关闭。

当前规则没有外部战略意图，因此暂不伪造没有差异来源的“时间线分支”。后续加入可审计的恒星干预意图后，分支必须保存意图日志，并从干预前的可信 tick 重建。

## 5. 纵切验收

- 同一 spec 和 seed 的单次推进、分批推进与恢复重放完全一致；
- 模型中的 `nebulaMass + stellarMass + expelledMatter` 全程守恒；
- 必须经历点燃、稳定燃烧结束、爆发和行星盘四个里程碑；
- 用户可以暂停、单步、选择倍速和恢复本地记录；
- Babylon 只从控制器快照派生星云、星体、爆发和行星盘效果；
- Babylon 星辰页保持实验状态，默认 Pages 构建不得包含入口、页面、内容 JSON 或 bundle；
- 关闭或不支持 WebGL 时，CLI 和规则测试仍可完成整个生命周期。

## 6. 后续扩展

1. 为恒星控制增加正式 ESIP schema 和持久 sidecar 调度；
2. 增加受限干预意图与可信分支；
3. 增加不同质量、角动量和结局的原创预设；
4. 第6篇原典确定后，再定义行星形成的下一份内容 schema。
